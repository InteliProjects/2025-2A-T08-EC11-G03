# Privacidade e proteção de dados

## Introdução

A proteção das informações geradas pelo sistema é fundamental para garantir a segurança da fauna monitorada e a integridade das ações de conservação. Embora não sejam coletados dados pessoais humanos, as informações ambientais e geográficas obtidas pelas coleiras têm alto grau de sensibilidade, pois podem revelar a localização de espécies ameaçadas e áreas protegidas. A gestão cuidadosa desses dados é indispensável para evitar riscos como caça ilegal, tráfico de fauna e degradação de habitats.

## Natureza e classificação dos dados

O sistema não coleta dados pessoais humanos, mas lida com informações sensíveis ambientais:

- Localização geográfica precisa de fauna silvestre.
- Eventos críticos (inatividade, deslocamento anormal).

Risco associado: divulgação indevida de coordenadas de espécies ameaçadas pode facilitar caça, tráfico de fauna e invasão de áreas protegidas.

**Classificação:** Dados não pessoais, mas com nível de proteção como a de dados críticos, para isso, será seguido a LGPD e boas práticas de dados sensíveis ambientais (UNEP Data Guidelines, 2022).

## Armazenamento de dados

- Criptografia aplicada tanto no armazenamento local quanto durante a transmissão para os servidores.
- Anonimização geográfica sempre que possível, especialmente para dados de espécies ameaçadas, aplicando técnicas de geo-masking em registros de localização.
- Controle de acesso baseado em perfis, garantindo que apenas profissionais autorizados, como administradores de pesquisa e segurança ambiental, possam visualizar dados completos.

## Uso dos dados

Os dados serão utilizados exclusivamente para:
- Monitoramento científico.
- Planejamento e execução de ações de conservação.
- Geração de relatórios.
Restrições:
- Divulgação de coordenadas exatas em material público.
- Proibida a comercialização dos dados ou seu uso para fins não relacionados à conservação.

## Práticas para garantir conformidade
- Aplicação dos princípios de minimização, finalidade e necessidade previstos na LGPD.
- Auditorias internas e externas anuais para verificar segurança e governança dos dados.
- Revisão periódica das políticas de dados conforme atualizações legais e recomendações internacionais.
- Treinamentos obrigatórios para administradores e operadores do sistema.

## Conclusão
As práticas descritas garantem que as informações sensíveis geradas pelo sistema sejam protegidas contra acessos indevidos e utilizadas exclusivamente para fins de conservação e pesquisa. Dessa forma, o projeto assegura que a coleta e o uso dos dados contribuam para a proteção da fauna e a credibilidade das ações realizadas.
